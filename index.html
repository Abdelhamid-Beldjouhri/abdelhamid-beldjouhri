Cybersecurity Incident Report

Status: Resolved Type: DoS / SYN Flood Date: 2026-05-22

// Scenario Summary

You work as a security analyst for a travel agency. Employees regularly access the company's sales webpage to search for vacation packages. One afternoon, an automated alert indicated a problem with the web server — attempting to visit the site returned a connection timeout error.

Using a packet sniffer, you captured a large number of TCP SYN requests coming from an unfamiliar IP address. The web server was overwhelmed by incoming traffic and losing its ability to respond. You took the server offline temporarily, blocked the suspicious IP, and prepared this report to inform your manager.

SECTION 01

Identify the Type of Attack

One potential explanation for the connection timeout error

The web server is being overwhelmed by an abnormally large volume of incoming TCP SYN requests from an unfamiliar IP address, preventing it from responding to legitimate traffic.

The logs show that

A large number of TCP SYN packets are originating from a single, unfamiliar IP address, far exceeding normal traffic levels. The server is losing its ability to respond to these requests as its resources become exhausted.

This event could be

⚠ SYN Flood Attack — Denial of Service (DoS)

A SYN flood attack is a type of Denial of Service (DoS) attack where a malicious actor floods the target server with TCP SYN packets to exhaust its connection resources, preventing it from accepting legitimate connections.

SECTION 02

How the Attack Is Causing the Website to Malfunction

The TCP Three-Way Handshake — Three Steps

  • 1 SYN — The client sends a SYN (synchronize) packet to the server to initiate a connection request.
  • 2 SYN-ACK — The server responds with a SYN-ACK (synchronize-acknowledge) packet, reserving resources and acknowledging the request.
  • 3 ACK — The client sends a final ACK (acknowledge) packet back to the server, completing the connection.

What happens when a malicious actor sends a large number of SYN packets at once

The server receives the flood of SYN packets and responds to each one with a SYN-ACK, reserving memory and connection-table entries for each half-open connection. However, since the attacker never completes the handshake — the final ACK is never sent — the server's connection table fills up entirely with these dangling half-open entries. This exhausts available resources and prevents the server from accepting any new, legitimate connections.

What the logs indicate and how that affects the server

The logs show an abnormally high volume of SYN requests originating from a single, unfamiliar IP address — a clear indicator of a SYN flood attack in progress. As a result of this attack, the server becomes overwhelmed, loses its ability to process incoming traffic from legitimate users, and begins returning connection timeout errors across the board — effectively taking the company's website offline.
